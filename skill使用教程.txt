========================================
            SKILL 使用教程
========================================

本教程包含所有小说创作相关智能体的使用说明。

========================================
1. novel-master-orchestrator - 小说创作主控智能体
========================================
描述：整合所有小说创作智能体的主控智能体，协调整体创作流程。
何时调用：需要全方位小说创作支持、从零开始创作小说、续写已有故事、整合SCP和崩坏3素材、或需要协调多个智能体协作时。

功能：
- 智能识别用户需求，自动调度相应专业智能体
- 协调整个创作流程，从资料整理到最终输出
- 管理不同智能体之间的信息传递和协作
- 提供三种工作模式：全新小说创作、续写已有小说、混合创作（SCP+崩坏3）
- 在关键节点自动进行质量控制和连贯性检查
- 输出完整的创作项目报告

整合的智能体：
1. deepseek-html-processor - DeepSeek TXT文件处理
2. scp-extractor - SCP文档提取
3. scp-archiver - SCP文档归档
4. storyline-organizer - 故事线整理
5. honkai-character-integrator - 崩坏3角色整合
6. character-scp-integrator - 角色与SCP能力整合
7. lore-organizer - 资料整理
8. story-writer - 小说写作
9. novel-continuer - 小说续写
10. coherence-checker - 连贯性检查
11. setting-reader - 设定阅读
12. html-to-pdf-converter - HTML转PDF转换

使用示例：
- "我想创作一部融合SCP基金会和崩坏3的小说，主角是琪亚娜，拥有类似SCP-3812的能力"
- "基于目录下的txt文件，继续创作故事"
- "帮我完善角色信息.txt中的角色设定，参考SCP-6820的能力"


========================================
2. setting-reader - 设定阅读智能体
========================================
描述：在写作前阅读SCP_PDF、崩坏3角色、角色等目录下的所有资料，整理设定信息，确保创作符合设定要求，防止设定走偏。
何时调用：创作小说前需要先阅读和整理所有设定资料时。

功能：
- 扫描和读取 e:\小说\SCP_PDF 目录下的所有SCP文档
- 扫描和读取 e:\小说\崩坏3角色 目录下的所有角色资料
- 扫描和读取 e:\小说\角色 目录下的角色信息.txt文件
- 读取 e:\小说\一起讨论故事想法 和 e:\小说\一些新设定或是其他的东西 目录下的内容
- 整理和结构化所有设定信息，为创作提供参考
- 确保创作符合已有设定，防止设定走偏
- 提供设定摘要和关键信息提醒

支持的SCP文档：
- SCP-3812 - SCP基金会_最终版.pdf
- SCP-6183 - SCP基金会.pdf
- SCP-6488 - SCP基金会.pdf
- SCP-6659 - SCP基金会.pdf
- SCP-6747 - SCP基金会.pdf
- SCP-6820 - SCP基金会.pdf
- SCP-7528 - SCP基金会.pdf
- 助理研究员Picsell Dois的演绎部研讨发言：三明治，叙事的稳定和填充 - SCP基金会.pdf
- 基金会里的神学 - SCP基金会.pdf
- 超形上学导论 - SCP基金会.pdf
- 述叙事学与你：摒弃虚构与非虚构之分 - SCP基金会.pdf

支持的崩坏3角色：
- 嗨♪爱愿妖精♥_百度百科.pdf
- 始源之律者_百度百科.pdf
- 死生之律者_百度百科.pdf
- 真我·人之律者_百度百科.pdf
- 真理之律者_百度百科.pdf
- 终焉之律者（游戏《崩坏3》中的角色）_百度百科.pdf

使用示例：
- "在开始创作前，请先阅读所有设定资料"
- "请帮我查阅终焉之律者和山天远的设定"
- "我要写一章，先帮我看看需要注意什么设定"


========================================
3. deepseek-html-processor - DeepSeek TXT处理器
========================================
描述：从DeepSeek保存的TXT文件中提取对话内容、角色设定、章节文本等信息，供其他智能体使用。
何时调用：需要处理DeepSeek保存的聊天记录TXT文件时。

功能：
- 提取用户与AI的完整对话历史
- 识别和分离角色设定、章节内容、故事大纲等不同类型的信息
- 支持批量处理多个TXT文件
- 输出结构化的纯文本内容供其他智能体使用

处理内容类型：
- 对话历史提取
- 角色设定识别
- 故事内容提取
- 其他创作资料

使用示例：
- "处理崩坏3故事1.txt"
- "处理与deepseek一起创作小说章节目录下所有TXT文件"
- "从TXT中提取所有角色信息"


========================================
4. storyline-organizer - 故事线整理智能体
========================================
描述：提取 e:\小说\与deepseek一起创作小说章节 中所有txt文件并整理完整故事线，帮助用户彻底了解故事发展。
何时调用：需要整理故事线、回顾剧情或了解故事全貌时。

功能：
- 扫描并读取 e:\小说\与deepseek一起创作小说章节 目录下所有txt文件
- 按章节顺序整理完整故事内容
- 提取关键剧情节点和人物发展轨迹
- 生成故事概要和时间线
- 帮助用户快速了解故事全貌和发展脉络

输出内容：
- 完整故事简介
- 每章核心内容摘要
- 主要人物发展轨迹
- 关键事件时间线
- 故事主题分析

使用示例：
- "请帮我整理所有章节的故事线"
- "帮我整理第5-10章的故事发展"
- "帮我追踪某角色在所有章节中的发展"


========================================
5. scp-extractor - SCP文档提取器
========================================
描述：读取 e:\小说\SCP_PDF 中PDF文档，提取纯文本内容，节省Token消耗。
何时调用：需要处理SCP PDF文件或提取文档文本内容时。

功能：
- 从 e:\小说\SCP_PDF 目录下的 SCP 基金会 PDF 文档中智能提取纯文本内容
- 去除不必要的格式元素，节省 token 消耗
- 提取 PDF 中的所有文本内容
- 去除 PDF 格式标签和样式
- 保持文本结构和段落
- 支持批量处理多个 PDF 文件

使用示例：
- "提取 SCP-6820 - SCP基金会.pdf 的内容"
- "批量处理 e:\小说\SCP_PDF 中的所有 PDF 文件"


========================================
6. scp-archiver - SCP文档归档器
========================================
描述：管理和归档多个SCP文档，建立知识库索引，支持按编号、等级、标签等检索。
何时调用：需要整理多个SCP文档或建立知识库时。

功能：
- 自动扫描目录中的SCP HTML文件
- 提取文档元数据（编号、等级、标签等）
- 建立可搜索的索引数据库
- 支持按多种条件检索文档
- 生成文档摘要和关键词

检索功能：
- 按编号范围检索（如SCP-1000到SCP-2000）
- 按收容等级筛选（Safe/Euclid/Keter/Thaumiel等）
- 按关键词搜索
- 按标签分类浏览

使用示例：
- "扫描当前目录所有SCP HTML文件"
- "建立SCP知识库"


========================================
7. honkai-character-integrator - 崩坏3角色整合器
========================================
描述：处理崩坏3角色文件夹中的PDF文件，提取角色信息并将它们完全融入故事世界观中。
何时调用：需要整合崩坏3角色到故事中时。

功能：
- 读取和解析崩坏3角色文件夹中的PDF文件
- 提取角色的完整信息（背景、能力、性格、外貌等）
- 将崩坏3角色自然融入现有的故事世界观
- 建立崩坏3角色与原创角色的关系网
- 设计角色在故事中的定位和作用
- 确保角色融合的合理性和连贯性

崩坏3角色列表：
1. 终焉之律者 - 终结一切的律者
2. 真理之律者 - 掌握真理的律者
3. 始源之律者 - 起源的律者
4. 死生之律者 - 掌控生死的律者
5. 真我·人之律者 - 代表人性的律者
6. 嗨♪爱愿妖精♥ - 特殊角色

使用示例：
- "将终焉之律者融入故事"
- "处理崩坏3角色文件夹中的所有角色"
- "设计终焉之律者与山天远的相遇场景"


========================================
8. character-scp-integrator - 角色与SCP能力整合器
========================================
描述：处理角色信息.txt文件，将角色设定与SCP文档中的能力细节相结合，确保设定一致性和合理性。
何时调用：需要整合角色与SCP能力、检查设定匹配度时。

功能：
- 读取和解析角色信息.txt文件
- 查阅对应的SCP文档获取详细能力描述
- 将角色设定与SCP能力深度融合
- 确保能力描述的一致性和合理性
- 避免设定矛盾和能力不匹配
- 生成完整的角色能力档案

核心功能：
- 角色信息解析
- SCP能力对接
- 能力整合与优化
- 设定一致性检查

使用示例：
- "整合山天远的角色设定与SCP-3812"
- "处理角色信息.txt中的所有角色"
- "检查所有角色的能力是否平衡"


========================================
9. lore-organizer - 资料整理智能体
========================================
描述：整理和管理小说设定资料，包括角色档案、世界观、背景故事、时间线等。
何时调用：需要创建、整理或查阅小说设定资料时。

功能：
- 创建和管理角色档案
- 构建世界观和设定体系
- 整理背景故事和历史
- 建立时间线和关系图
- 生成设定文档和参考资料

管理内容：
- 角色档案
- 世界观设定
- SCP基金会专项设定
- 故事资料

使用示例：
- "基于SCP-6820创建角色和世界观设定"
- "整理已有设定资料"
- "查阅角色或世界观设定"


========================================
10. story-writer - 小说写作智能体
========================================
描述：专门负责撰写小说内容，基于整理好的资料和设定创作章节、段落和对话。
何时调用：需要写小说内容、创作章节或扩展故事时。

功能：
- 根据故事大纲和设定创作章节
- 撰写生动的对话和描写
- 控制故事节奏和情节发展
- 保持人物性格一致性
- 生成多个版本供选择

写作风格：
- 严肃文学风格
- 轻松幽默风格
- 悬疑惊悚风格
- 科幻奇幻风格
- SCP基金会风格（特别优化）

使用示例：
- "基于SCP-6820创作一篇基金会特工的故事"
- "创作第一章内容"


========================================
11. novel-continuer - 小说续写智能体
========================================
描述：专门用于继续DeepSeek已经创作的小说，保持故事连贯性和人物性格一致性。
何时调用：需要继续创作已有小说、续写章节或扩展故事时。

功能：
- 基于已有故事内容继续创作新章节
- 保持故事连贯性和人物性格一致性
- 遵循原有写作风格和叙事节奏
- 与deepseek-html-processor配合处理txt文件中的历史内容
- 与coherence-checker配合确保续写内容与原作一致

核心能力：
- 故事连贯性维护
- 写作风格保持
- 人物性格一致性

使用示例：
- "基于崩坏3故事12.txt继续创作下一章"
- "处理与deepseek一起创作小说章节目录下的所有txt文件，继续创作"
- "基于已有内容创作两个不同方向的版本"


========================================
12. coherence-checker - 连贯性检查智能体
========================================
描述：监督故事连贯性和合理性，检查人物性格一致性、情节逻辑、时间线等问题。
何时调用：写好小说内容后需要审核、发现矛盾或改进故事时。

功能：
- 检查人物性格是否一致
- 验证情节逻辑是否合理
- 追踪时间线是否连续
- 发现设定矛盾和bug
- 提供改进建议和修改方案

检查项目：
- 人物一致性检查
- 情节逻辑检查
- 时间线检查
- 设定一致性检查

使用示例：
- "审核刚写的小说章节"
- "检查故事是否有矛盾"


========================================

========================================
描述：将HTML文件1:1精确转换为PDF格式，保留所有样式和布局。
何时调用：用户发出'我想使用'指令时。

功能：
- 精确1:1转换HTML文件为PDF格式
- 保留所有CSS样式、布局和格式
- 支持图片、表格、链接等所有HTML元素
- 保持页面尺寸和分页正确
- 支持多页HTML文档转换

使用示例：
- "将网页保存为PDF"
- "转换HTML报告为PDF"


========================================
            使用建议
========================================

推荐工作流程：

1. 全新创作：
   - 调用 novel-master-orchestrator（主控）
   - 或者依次调用：setting-reader → lore-organizer → story-writer → coherence-checker

2. 续写故事：
   - 调用 novel-master-orchestrator（主控）
   - 或者依次调用：deepseek-html-processor → setting-reader → lore-organizer → novel-continuer → coherence-checker

3. 混合创作（SCP+崩坏3）：
   - 调用 novel-master-orchestrator（主控）
   - 或者依次调用：setting-reader → scp-extractor → honkai-character-integrator → character-scp-integrator → lore-organizer → story-writer → coherence-checker

最简单的方式：直接使用 novel-master-orchestrator，它会自动协调所有其他智能体！
